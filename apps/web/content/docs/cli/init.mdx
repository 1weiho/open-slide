---
title: open-slide init
description: Scaffold a new workspace.
---

```npm
npx @open-slide/cli init [directory]
```

If `directory` is omitted, the scaffolder uses the current folder. If it's
provided and doesn't exist, it's created.

## What it generates

```text
my-deck/
├── slides/
│   └── getting-started/
│       └── index.tsx       # starter slide
├── themes/                 # empty
├── open-slide.config.ts    # framework config
├── CLAUDE.md               # mirrors AGENTS.md
├── AGENTS.md               # agent rules
├── package.json
├── tsconfig.json
└── netlify.toml / vercel.json   # one-click deploy configs
```

## After init

```npm
cd my-deck
npm install
npm run dev
```

The dev server boots, opens the slide browser, and you're ready for your
agent to take over.
